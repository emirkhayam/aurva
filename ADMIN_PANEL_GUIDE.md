# 🎯 AURVA Admin Panel - Руководство

## ✅ Что создано

Полностью функциональная админ-панель для управления контентом сайта AURVA.

### 📦 Технологический стек

- **Frontend Framework**: React 18 + TypeScript
- **Build Tool**: Vite (быстрая сборка и dev сервер)
- **Styling**: Tailwind CSS (дизайн в стиле основного сайта AURVA)
- **UI Components**: Кастомные компоненты в стиле shadcn/ui
- **Routing**: React Router v6
- **State Management**: Zustand
- **HTTP Client**: Axios
- **Icons**: Lucide React
- **Notifications**: Sonner (toast)

### 🎨 Дизайн

Админ-панель полностью соответствует дизайн-системе основного сайта AURVA:

- ✅ Цветовая схема AURVA (синий акцент #2563EB)
- ✅ Шрифты: Oswald (заголовки) + Inter (текст)
- ✅ Минималистичный, чистый дизайн
- ✅ Тонкие тени и blur эффекты
- ✅ Профессиональный корпоративный стиль

### 🚀 Функционал

#### 1. **Аутентификация**
- JWT авторизация
- Защищенные роуты
- Автоматический logout при истечении токена
- Страница входа с валидацией

#### 2. **Дашборд**
- Статистика по заявкам (всего, новых)
- Статистика по новостям (всего, опубликованных)
- Статистика по участникам (всего, активных)
- Обзор системы

#### 3. **Управление заявками (Contacts)**
- Просмотр всех заявок на вступление
- Фильтрация по статусу:
  - Новая
  - Связались
  - Обработана
  - Отклонена
- Изменение статуса заявок
- Удаление заявок
- Цветные badges для статусов

#### 4. **Управление новостями (News)**
- CRUD операции (создание, чтение, обновление, удаление)
- Категории:
  - Регулирование
  - События
  - Аналитика
  - Другое
- Загрузка изображений для новостей
- Публикация/скрытие новостей (drafts)
- Slug автогенерация на backend
- Поля:
  - Заголовок
  - Краткое описание
  - Полный текст
  - Категория
  - Изображение
  - Статус публикации

#### 5. **Управление участниками (Members)**
- CRUD операции для участников ассоциации
- Загрузка логотипов компаний
- Активация/деактивация участников
- Управление порядком отображения (displayOrder)
- Поля:
  - Название компании
  - Описание
  - Веб-сайт (с валидацией URL)
  - Логотип
  - Дата вступления
  - Статус (активный/неактивный)
  - Порядок отображения

#### 6. **Профиль (Profile)**
- Просмотр информации пользователя
- Изменение пароля
- Информация о роли (admin/moderator)
- ID пользователя

#### 7. **Навигация и Layout**
- Сайдбар с навигацией
- Логотип AURVA
- Информация о пользователе в сайдбаре
- Кнопка выхода
- Адаптивный дизайн

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
cd admin-panel
npm install
```

### 2. Настройка окружения

Файл `.env` уже создан с правильными настройками:

```env
VITE_API_URL=http://localhost:5000/api
```

Убедитесь, что backend API запущен на порту 5000.

### 3. Запуск backend API (если еще не запущен)

В отдельном терминале:

```bash
cd backend
npm run dev
```

Backend должен работать на `http://localhost:5000`

### 4. Запуск админ-панели

```bash
cd admin-panel
npm run dev
```

Админ-панель будет доступна по адресу: **http://localhost:5173**

### 5. Вход в систему

Используйте учетные данные администратора из backend:

**Логин:**
- Email: `admin@aurva.kg` (или из backend `.env` ADMIN_EMAIL)
- Пароль: пароль, установленный при `npm run seed:admin` в backend

**Если администратор не создан:**

```bash
cd backend
npm run seed:admin
```

## 📖 Использование

### Работа с заявками

1. Перейдите в раздел **"Заявки"**
2. Увидите список всех заявок с фильтрами
3. Для каждой заявки можно:
   - Изменить статус через выпадающий список
   - Удалить заявку (кнопка с корзиной)
4. Фильтруйте заявки по статусу через кнопки сверху

### Работа с новостями

1. Перейдите в раздел **"Новости"**
2. Нажмите **"Добавить новость"** для создания
3. Заполните форму:
   - Заголовок (обязательно)
   - Краткое описание (обязательно)
   - Полный текст (обязательно)
   - Категория (выбор из списка)
   - Изображение (опционально)
   - Чекбокс "Опубликовать сразу"
4. Для редактирования существующей новости нажмите **"Редактировать"**
5. Для удаления нажмите кнопку с корзиной

### Работа с участниками

1. Перейдите в раздел **"Участники"**
2. Нажмите **"Добавить участника"** для создания
3. Заполните форму:
   - Название компании (обязательно)
   - Описание (обязательно)
   - Веб-сайт (URL, опционально)
   - Дата вступления (обязательно)
   - Порядок отображения (число для сортировки)
   - Логотип (изображение, опционально)
   - Чекбокс "Активный участник"
4. Для редактирования нажмите **"Редактировать"**
5. Для удаления нажмите кнопку с корзиной

### Изменение пароля

1. Перейдите в раздел **"Профиль"**
2. Введите текущий пароль
3. Введите новый пароль (минимум 6 символов)
4. Подтвердите новый пароль
5. Нажмите **"Изменить пароль"**

## 🌐 API интеграция

Админ-панель взаимодействует с backend API через настроенный Axios instance.

### Базовый URL

`http://localhost:5000/api` (настраивается в `.env`)

### Автоматические функции

- **Добавление токена**: Axios автоматически добавляет JWT токен в заголовки
- **Обработка 401**: Автоматический logout и редирект на /login
- **Обработка ошибок**: Toast уведомления об ошибках

### Используемые endpoints

**Auth:**
- `POST /auth/login` - вход
- `GET /auth/profile` - профиль
- `PUT /auth/change-password` - смена пароля

**Contacts:**
- `GET /contacts` - список заявок
- `PUT /contacts/:id` - обновление статуса
- `DELETE /contacts/:id` - удаление

**News:**
- `GET /news` - список новостей
- `POST /news` - создание (multipart/form-data)
- `PUT /news/:id` - обновление (multipart/form-data)
- `DELETE /news/:id` - удаление

**Members:**
- `GET /members` - список участников
- `POST /members` - создание (multipart/form-data)
- `PUT /members/:id` - обновление (multipart/form-data)
- `DELETE /members/:id` - удаление

## 🚢 Production Deployment

### 1. Build

```bash
cd admin-panel
npm run build
```

Создаст оптимизированную версию в папке `dist/`

### 2. Preview локально

```bash
npm run preview
```

### 3. Deploy

Админ-панель - это статическое SPA приложение. Можно разместить на:

- **Vercel** (рекомендуется) ← один клик deploy
- Netlify
- GitHub Pages
- Cloudflare Pages
- Любой статический хостинг

### 4. Production настройки

**Важно перед деплоем:**

1. Обновите `.env` для production:
   ```env
   VITE_API_URL=https://your-backend-domain.com/api
   ```

2. На backend обновите CORS:
   ```env
   CORS_ORIGIN=https://your-admin-panel-domain.com
   ```

3. Убедитесь, что backend доступен по HTTPS

### Deploy на Vercel (рекомендуется)

```bash
# Установите Vercel CLI
npm i -g vercel

# Deploy
cd admin-panel
vercel
```

Или через GitHub integration - push в репозиторий и автоматический deploy.

## 🐛 Troubleshooting

### Проблема: "Failed to fetch" при входе

**Решение:**
1. Убедитесь, что backend запущен на `http://localhost:5000`
2. Проверьте `.env` файл - должен быть `VITE_API_URL=http://localhost:5000/api`
3. Проверьте CORS настройки в backend

### Проблема: "Invalid credentials" при входе

**Решение:**
1. Убедитесь, что администратор создан: `cd backend && npm run seed:admin`
2. Проверьте email и пароль в `backend/.env`

### Проблема: Изображения не загружаются

**Решение:**
1. Убедитесь, что папка `backend/uploads` существует
2. Проверьте права доступа к папке
3. Backend должен раздавать статические файлы из `/uploads`

### Проблема: Не работает после build

**Решение:**
- Убедитесь, что в `.env` правильный production API URL
- Настройте redirects для SPA (все → index.html)
