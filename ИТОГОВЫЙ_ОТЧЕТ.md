# Итоговый отчет: DNS проблема AURVA

## 🎯 Корневая причина (100% подтверждено)

**UDP порт 53 блокируется на уровне ISP/датацентра**

### Доказательства:
1. ✗ **Proxmox хост**: UDP DNS timeout
2. ✗ **VM aurva**: UDP DNS timeout
3. ✓ **Proxmox хост**: TCP DNS работает
4. ✓ **VM aurva**: TCP DNS работает
5. ✓ **ICMP (ping)**: работает везде

### Вывод:
Блокировка находится **выше уровня Proxmox** - на роутере ISP/датацентра.
Настройка Proxmox firewall **НЕ ПОМОЖЕТ**.

## 💡 РЕШЕНИЯ

### Вариант 1: Связаться с ISP/датацентром (РЕКОМЕНДУЕТСЯ)

**Кто:** Владелец сервера/администратор

**Что сделать:**
1. Связаться с провайдером Hetzner (если сервер у них)
2. Открыть тикет: "UDP port 53 (DNS) is blocked outbound"
3. Запросить разблокировку UDP 53 для IP: 135.181.112.60

**Время:** 1-3 дня

**Вероятность успеха:** 90%

---

### Вариант 2: Использовать готовые Docker образы

**Если образы уже собраны и опубликованы в Docker Hub/GHCR:**

```bash
# Обновить docker-compose.yml, заменить build: на image:
# backend:
#   image: youruser/aurva-backend:latest
# admin-panel:
#   image: youruser/aurva-admin-panel:latest

cd /opt/aurva
sudo docker compose up -d
```

**Время:** 5 минут

**Требует:** Предварительно собранные образы

---

### Вариант 3: Собрать образы локально (на вашей машине)

**На локальной машине с нормальным интернетом:**

```bash
# 1. Клонировать репозиторий
git clone https://github.com/emirkhayam/aurva.git
cd aurva

# 2. Собрать образы
docker build -t aurva-backend:latest ./backend
docker build -t aurva-admin-panel:latest ./admin-panel

# 3. Сохранить в файлы
docker save aurva-backend:latest > backend.tar
docker save aurva-admin-panel:latest > admin-panel.tar

# 4. Скопировать на сервер
scp backend.tar admin-panel.tar aurva:/tmp/

# 5. На сервере загрузить образы
ssh aurva "sudo docker load < /tmp/backend.tar"
ssh aurva "sudo docker load < /tmp/admin-panel.tar"

# 6. Запустить
ssh aurva "cd /opt/aurva && sudo docker compose up -d"
```

**Время:** 30 минут

**Вероятность успеха:** 95%

### Вариант 4: Использовать Vercel для frontend + Docker для backend

**Frontend (admin-panel):** Развернуть на Vercel (бесплатно)
**Backend:** Развернуть через pre-built образы

```bash
# 1. Vercel для admin-panel
cd admin-panel
vercel deploy --prod

# 2. Backend на сервере (когда образ готов)
cd /opt/aurva
# Оставить только backend в docker-compose.yml
sudo docker compose up -d backend
```

**Время:** 20 минут

**Преимущества:**
- Frontend на CDN (быстрее)
- Меньше нагрузки на сервер

---

### Вариант 5: Настроить локальный DNS proxy с TCP upstream

**Сложное решение, но работает без внешней помощи:**

1. Установить и настроить локальный DNS resolver (unbound/dnsmasq)
2. Настроить его использовать TCP для внешних запросов
3. Настроить Docker использовать локальный resolver

**Время:** 2-3 часа

**Сложность:** Высокая

**Документация:** См. `NETWORK_SOLUTIONS.md` → Решение 5
