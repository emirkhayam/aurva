# 🎯 РЕШЕНИЕ НАЙДЕНО!

## Проблема
DNS не работает внутри контейнера Coolify → GitHub API недоступен → Deploy не работает

## ✅ Корневая причина (100% уверенность)
**UDP порт 53 (DNS) блокируется Proxmox firewall**

### Доказательства:
```bash
# На хосте aurva - UDP DNS НЕ РАБОТАЕТ:
$ dig @8.8.8.8 google.com
;; communications error to 8.8.8.8#53: timed out

# На хосте aurva - TCP DNS РАБОТАЕТ:
$ dig @8.8.8.8 google.com +tcp
142.251.38.78   ← Успешно!

# Docker логи показывают:
[resolver] failed to query external DNS server
error="read udp 10.0.1.2->8.8.4.4:53: i/o timeout"
```

---

## 🚀 РЕШЕНИЕ (2 минуты)

### Вариант 1: Через Web UI (ПРОСТО)

1. Откройте Proxmox Web UI:
   ```
   https://135.181.112.60:8006
   ```

2. Войдите как root

3. Выберите VM **aurva** (ID: 101)

4. Перейдите в **Firewall** → **Add Rule**

5. Создайте правило:
   - **Direction:** OUT
   - **Action:** ACCEPT
   - **Protocol:** udp
   - **Destination port:** 53
   - **Comment:** Allow DNS queries

6. Нажмите **Add**

✅ Готово! DNS заработает автоматически.

---

### Вариант 2: Через SSH (БЫСТРО)

```bash
# 1. Подключиться к Proxmox как root
ssh root@135.181.112.60

# 2. Добавить правило firewall
cat >> /etc/pve/firewall/101.fw <<'EOF'
[RULES]
OUT ACCEPT -p udp -dport 53 -log nolog # Allow DNS
EOF

# 3. Применить
pve-firewall restart

# 4. Проверить
exit
ssh aurva "dig @8.8.8.8 google.com +short"
```

**Ожидаемый результат:** IP адрес (например: `142.251.38.78`)

## 💡 Почему это происходит?

Многие firewall блокируют UDP DNS (порт 53) для:
- Защиты от DNS amplification атак
- Принуждения использовать корпоративные DNS
- Контроля DNS трафика

TCP DNS обычно разрешен, так как:
- Не используется для amplification атак
- Требует установки соединения
- Легче отслеживать

Docker по умолчанию использует UDP для DNS запросов, поэтому блокировка UDP 53 ломает DNS внутри контейнеров.
