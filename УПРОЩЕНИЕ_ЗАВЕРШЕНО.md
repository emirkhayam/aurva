## 🚀 Как запустить

### Локальная разработка

```bash
# 1. Создать .env
cp .env.example .env

# 2. Запустить
docker-compose -f docker-compose.dev.yml up --build

# 3. Открыть
# 🌐 http://localhost:3000 - публичный сайт
# 🔐 http://localhost:3000/admin - админка
# 🔌 http://localhost:3000/api - API
```

**Один сервис, один порт, одна команда!**

### Production

```bash
# На сервере (Coolify)
docker-compose up --build -d
```

## ✏️ Как редактировать

### 1. Публичный сайт (HTML)

```bash
# Редактируйте в backend/public/
nano backend/public/index.html

# Изменения видны сразу на localhost:3000
# (благодаря volume mount в docker-compose)
```

### 2. React админка

```bash
# Разработка
cd admin-panel
nano src/App.tsx

# Сборка
npm run build

# Копирование в backend
cp -r dist/* ../backend/public/admin/
```

### 3. Backend API

```bash
# Редактируйте
nano backend/src/routes/newsRoutes.ts

# Nodemon автоматически перезагрузит
```

## 📊 Сравнение

### Docker Compose

**Было (v2.0):**
```yaml
services:
  nginx:         # Reverse proxy
  backend:       # API
  admin-panel:   # React dev server
```

**Стало (v3.0):**
```yaml
services:
  backend:       # Все в одном!
```

### URL Routing

**Было:**
```
http://localhost        → nginx → frontend/
http://localhost/admin  → nginx → admin-panel:5173
http://localhost/api    → nginx → backend:3000
```

**Стало:**
```
http://localhost:3000/       → backend → public/index.html
http://localhost:3000/admin  → backend → public/admin/
http://localhost:3000/api    → backend → routes/
```

### Код (backend/src/app.ts)

**Было:**
```typescript
// Root endpoint
app.get('/', (_req: Request, res: Response) => {
  res.json({
    message: 'AURVA Backend API',
    endpoints: { ... }
  });
});
```

**Стало:**
```typescript
// API info endpoint
app.get('/api', (_req: Request, res: Response) => {
  res.json({
    message: 'AURVA Backend API',
    endpoints: { ... }
  });
});

// '/' теперь отдает index.html через express.static()
```

## ⚠️ Важные правила

### ✅ ДЕЛАТЬ:
1. Редактировать HTML **только** в `backend/public/`
2. Использовать `http://localhost:3000` для всего
3. Собирать админку перед деплоем: `npm run build`
4. Копировать build в `backend/public/admin/`

### ❌ НЕ ДЕЛАТЬ:
1. НЕ редактировать файлы в `frontend/` (deprecated!)
2. НЕ создавать дублирующиеся файлы
3. НЕ забывать собрать и скопировать админку
4. НЕ коммитить .env файлы
